# Issue Idea — 2026-03-14

Title: Add `account` field to `WalletConfig` to auto-import seed phrase during fixture setup

Summary:
`WalletConfig` already has a companion `WalletAccount` type defined in `types.ts`, but accounts are never linked to the wallet configuration — callers must manually call `wallet.importSeedPhrase()` in a `beforeAll` hook in every test file. Moving seed/account configuration into `WalletConfig` would allow chroma to auto-import the account during fixture initialization, eliminating repetitive boilerplate.

Why it matters:

* Every test file that uses chroma must duplicate a `beforeAll(() => wallet.importSeedPhrase({ seedPhrase }))` block, scattering seed phrases across test files and making global credential changes error-prone.
* There is no standard way to share a single account configuration across multiple spec files in a large test suite — the wallet type and its credentials currently live in two separate places.
* Worker-scoped fixture setup is the correct place for one-time wallet initialization, but chroma provides no declarative hook to do it automatically; the responsibility falls entirely on the test author.

Suggested scope:

* Add an optional `account?: WalletAccount` field to the `WalletConfig` interface in `types.ts`.
* In the `walletExtensionIds` (or a new dedicated worker fixture), detect when `account` is set and call the appropriate wallet import method automatically after extension IDs are resolved.
* Make the import idempotent — skip re-importing if the wallet already has accounts present (e.g., by checking for existing accounts in the extension popup before attempting import).

Category:
developer-experience

Optional notes:
This change would also enable a clean pattern where `account` is populated from environment variables at the `createWalletTest(...)` call site (e.g., `account: { seed: process.env.TEST_SEED_PHRASE! }`), rather than being hard-coded in every `beforeAll`. A follow-on enhancement could support an array of accounts for wallets that allow multiple imported accounts, such as Polkadot-JS or Talisman.
